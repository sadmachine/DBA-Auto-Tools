PO Screen Access Errors

  • Overview

Overview

If you receive a message from DBA when you attempt to open the PO Screen that says something along the lines of "A component named frmPurchase already exists" OR "Access Violation error in EJSME.EXE", this is actually a known issue with DBA Manufacturing, and doesn't have anything to do with the DBA AutoTools program.

DBA Manufacturing has a support article related to this issue, which can be found here.

The fix for this issue is essentially to make sure that the computer has a default printer assigned. If there is already a default printer assigned, try assigning a different printer as the default, then reassigning the original printer as the default again. You can find an article on how to assign default printers in Windows here.
