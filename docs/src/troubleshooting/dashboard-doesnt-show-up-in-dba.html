Dashboard Doesn't Show Up in DBA

Contents

  • Summary
  • Problem Description
  • Potential Causes/Solutions

Summary

  1. Verify that the required executables and shortcuts exist.
    • If either is missing, a reinstallation of the latest version is required.
      • If you know where to find the installer, you can run it yourself on the affected machine. The installer is usually located over the network, wherever the DBA Manufacturing installation is, within the Add-Ons\DBA AutoTools\Installers folder.
      • If you or the system administrator is unaware of where to find the installer for the latest version, please contact the original person responsible for installation, and let them know that a reinstallation is necessary.
  2. Verify that the program is running by checking for the Tray Icon and Processes that should exist.
    • If the Tray Icon is missing but the Processes are running, try ending the process and restarting the program from the Desktop Icon. Then, verify again that the program is running by checking for the Tray Icon and the running Processes.
    • If the Tray Icon is missing and the Processes do not exist, that means the program is not running and you must start the program from the Desktop Icon. Then, verify that the program is running by checking for the Tray Icon and the running Processes.
  3. If you have performed all of the steps above and the DBA AutoTools dashboard still is not showing, please contact the person responsible for installating the program for further troubleshooting.

Problem Description

As long as DBA AutoTools is running in the background, the expected behavior is that the DBA AutoTools dashboard will appear in the main DBA Manufacturing screen, like below:

If the DBA AutoTools dashboard does not appear as it does in the above image, that means something is not functioning as expected.

Potential Causes/Solutions

Cause: DBA AutoTools is not running

In order for the DBA AutoTools dashboard to show up, the program must be running in the background. You can verify whether or DBA AutoTools is running by checking for the Tray Icon as well as the Processes which indicate the programming is running properly.

If the Tray Icon is missing but the Processes are running, try closing the Processes from the Task Manager, and restarting the program from the shortcut that should exist on the desktop. Then verify that the Tray Icon exists, and that the DBA AutoTools dashboard is appearing properly within the DBA Manufacturing program.

If neither the Tray Icon nor the Processes exist, try launching the program from the icon on the desktop. Then, verify that the Tray Icon and Processes exists, and that the DBA AutoTools dashboard is appearing properly within the DBA Manufacturing program.

If neither of the above steps solves the problem, continue to the next section.

Cause: DBA AutoTools is not properly installed

In order for the DBA AutoTools dashboard to appear, the program has to be properly installed. If you tried the steps above, and the program would not start properly, this most likely means that there is a problem with the installation. Please review and follow any troubleshooting steps found in the Installation Checks, in particular the section on Executables.

Once you've reviewed the installation checks, attempt to open the DBA AutoTools program again from the desktop shortcut and verify that the Tray Icon and Processes exist. Then, verify that the DBA AutoTools dashboard shows up in the DBA Manufacturing screen.

If you've reached this point and the DBA AutoTools dashboard still isn't appearing properly, please contact the person responsible for installing the program originally to troubleshoot further.
