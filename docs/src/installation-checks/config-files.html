Summary

Verify that app/settings.ini (found within the installation directory) contains paths to valid directories for both local and global.

  • local should be a path on the local machine, most likely in the installation directory.
    (something like C:\DBA Help\DBA AutoTools\app\config)
  • global should be a path on the network.
    (something like \\server-name\DBA Manufacturing\Add-Ons\DBA AutoTools\config)

Introduction

Config files are used to store user editable settings that affect how the program operates at runtime. For instance, there is a config file which contains information about what the next inspection number should be when generating inspection documents. What config files/values exist depends on the modules used and the specific implementation of the program.

These files/values can be edited by using the supplied Settings.exe program, found within the main installation directory.

Often times, if a config file is missing, the program will notify you of that error and explain what needs to be done to fix the situation. Sometimes, the program will even force you to specify a value before you are able to continue using the program.

Types/Locations

There are two types of config files:

Local
Determines settings which will affect the current machine/installation only
These files are stored on the current machine
Global
Determines settings which will affect all machines/installations that are running DBA AutoTools
These files are stored on the network

The program determines where to find both local and global configuration files by reading information from the initialization file, app/settings.ini. If the settings.ini file specifies incorrect locations or files that do not exist, there may be some undefined behavior. Please verify that these locations seem correct.

An example of a valid app/settings.ini is below:


[location]
local=C:\DBA Help\DBA AutoTools\app\config\
global=\\stelray-erp\DBA Manufacturing\Add-Ons\DBA AutoTools\config
