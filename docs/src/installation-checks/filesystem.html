1) Installation Directory

The default installation directory for DBA AutoTools is C:\DBA Help\DBA AutoTools, if you do not find the installation at this location please consult with the person who installed the program to find out where it was installed.

2) Directories

Upon installation, DBA AutoTools creates a directory structure which is required for the program to function properly. Its important that the directory structure remains intact, and nothing gets deleted/rearranged.

The complete directory structure is found below. If any of the directories are missing, something is wrong and the program should either be reinstalled, or the missing directories need to be recreated: 


DBA AutoTools/
└── app/
    ├── config/
    ├── modules/
    ├── queue/
    ├── storage/
    │   └── logs/
    └── templates/

3) Executables

There are certain executable (.exe) files that must exist in the installation directory in order for the program to function properly. There are two executables that always must exist and allow the main dashboard to function (Main Executables), and then for each module there is an additional executable file that must be present (Module Executables).

3.1) Main Executables

The two main executables that must exist are DBA AutoTools.exe and QueueManager.exe. If either of these programs are missing, the main dashboard will not appear, and any document generation or printing operations that happen in the background will remain stuck in the queue. These executables must be found in the root of the installation directory, like below: 


DBA AutoTools/
├── ...
├── DBA AutoTools.exe
├── QueueManager.exe
└── ...

3.2) Module Executables

For each installed module, there is a corresponding executable file, found within the app/modules folder. So, if there are any buttons on the dashboard that appear not to be working, it might be because its executable is missing or was deleted.

An example
The dashboard contains three modules (buttons)

  • Job Receipts
  • PO Receipts
  • Stock Transfer

We would expect to see a file structure like the below: 


DBA AutoTools/
├── ...
├── app/
│   ├── ...
│   ├── modules/
│   │   ├── Job_Receipts.exe
│   │   ├── PO_Receipts.exe
│   │   └── Stock_Transfer.exe
│   └── ...
└── ...

4) Initialization Files

There are a set of files that are read at program startup and provide settings/context for how the program should operate within a given environment. These files are often called Initialization Files and have the extension .ini, although we will see that's not always the case. Without these files, the program will not function properly.

There are 3 required .ini files within the app directory, and one .env file within the root installation directory. The complete list of files is below. 


DBA AutoTools/
├── app/
│   ├── ...
│   ├── mods.ini
│   ├── settings.ini
│   └── version.ini
├── ...
├── .env
└── ...
